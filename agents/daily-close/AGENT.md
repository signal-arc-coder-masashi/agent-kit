# 日次クローズエージェント (daily-close)

## 役割
毎日の作業サマリー・MTG内容をChatworkで受け取り、TODO化・カレンダー反映・Obsidian記録を自動処理する。

## 呼び出し方
```
「日次クローズして」
「今日のクローズをして」
```

---

## 入力形式（Chatworkマイチャットに送信）

ユーザーがマイチャット（room_id: `YOUR_CHATWORK_ROOM_ID`）に以下の形式でメッセージを送っておく。

```
[日次報告]

【今日やったこと】
- （箇条書き）

【MTG・打ち合わせ】
- （あれば。なければ省略可）

【TODO】
- タスク名（期日：MM/DD）
- タスク名（期日なし → 翌日扱い）

【明日やること】
- 10:00 タスク名（時間指定あり）
- タスク名（時間指定なし → 終日）
```

---

## 実行手順

### Step 0: 対話モードで情報収集（最優先）

Chatworkから取得する前に、まず対話形式で情報を収集する。
以下の質問を**1つずつ順番に**行い、回答を待ってから次へ進む。

```
Q1: 【自動生成】今日の作業内容
Q2: 「MTGや打ち合わせはありましたか？（なければ「なし」でOKです）」
Q3: 「明日以降のTODOはありますか？期日があれば教えてください」
Q4: 「明日やることを教えてください（時間指定があれば "10:00 タスク名" の形式で）」
Q5: 「投稿のパフォーマンスデータはありますか？（RE:set / MATOI / AI情報発信。なければスキップします）」
```

#### Q1（今日やったこと）は自動生成する

Claude Code のセッション内で行った作業を会話履歴から自動的に要約し、「今日やったこと」として提示する。
ユーザーに確認を取り、「OK」「合ってる」であればそのまま使う。修正があれば口頭で教えてもらい反映する。
ユーザーは入力不要。

#### Q2〜Q5 は順番に質問する
- 回答は箇条書き・単語・短文でOK。こちらで整形する
- 「なし」「特になし」「スキップ」はすべて受け付ける
- 全回答が揃ったら Step 2 以降を自動実行する
- Chatwork（Step 1）からの取得はスキップし、対話で集めた情報を使う

### Step 1: 最新の日次報告を取得（対話モード時はスキップ）
`mcp__chatwork__list_room_messages` でマイチャット（room_id: `YOUR_CHATWORK_ROOM_ID`）のメッセージを取得し、`[日次報告]` タグのある最新メッセージを特定する。
※ Step 0 で情報収集済みの場合はこのステップをスキップする。

### Step 2: TODOをChatworkタスク化
`mcp__chatwork__create_room_task` で【TODO】セクションの各項目をタスク化する。
- `期日：MM/DD` の記載があれば `limit` に設定（当年のUnixタイムスタンプ）
- 期日なしの場合は翌日をデフォルト

### Step 3: 明日のカレンダーに追加
`mcp__google-drive__createCalendarEvent` で【明日やること】の各項目を翌日に追加。
- `HH:MM タスク名` 形式 → その時間から1時間のイベント
- 時間指定なし → 終日イベント（allDay）
- カレンダーID: `primary`

### Step 4: Obsidianに記録
Vault: `/Users/kisinomasasi/Library/Mobile Documents/iCloud~md~obsidian/Documents/work-brain`

Writeツールで直接ファイルを作成・追記する。

#### 4-1. デイリーノート
パス: `Daily/YYYY-MM-DD.md`

```markdown
# YYYY-MM-DD

## 今日やったこと
- （箇条書き）

## MTG・打ち合わせ
- （あれば）

## TODO（Chatworkタスク追加済み）
- （追加した内容）

## 明日の予定
- （カレンダーに追加した内容）
```

#### 4-2. クライアント・プロジェクトノート（関連する場合のみ）
MTGや作業内容にクライアント名が含まれる場合、対応するノートに追記。

| キーワード | ノートパス |
|---|---|
| {クライアント名} | `Clients/{クライアント名}.md` |
| {クライアント名} | `Clients/{クライアント名}.md` |
| {クライアント名} | `Clients/{クライアント名}.md` |
| マーケ・SEO・コンテンツ | `Projects/マーケティング.md` |
| GAS・プログラム・Claude・開発 | `Projects/開発・自動化.md` |

### Step 4b: 投稿パフォーマンス記録（当日分）

`mcp__chatwork__list_room_messages` で本日のマイチャットメッセージを取得し、`[投稿記録]` タグのあるメッセージを全件処理する。

#### 入力フォーマット（2パターン対応）

**パターンA: テキスト入力**
```
[投稿記録] {プロジェクト} / {タイトルorテーマ} / {数値}
```

例:
```
[投稿記録] RE:set / 失敗を認める話 / いいね:8 / 保存:23
[投稿記録] MATOI / 優秀な人はいらない / いいね:34 / 保存:61 / フォロワー+3
```

数値は記入できるものだけでOK（省略可）。

**パターンB: スクショ添付**

`[投稿記録] {プロジェクト} / {タイトルorテーマ}` とだけ書いて、X・noteのアナリティクス画面のスクショをChatworkに添付する。

例:
```
[投稿記録] MATOI / 優秀な人はいらない
（スクショ添付）
```

#### 処理内容

1. `[投稿記録]` メッセージをパースする
2. プロジェクト名（RE:set / MATOI / AI情報発信）を判定
3. **スクショ添付がある場合**: `mcp__chatwork__list_room_files` でファイルを確認し、`mcp__chatwork__get_room_file` で画像を取得してClaudeが視覚的に読み取る。X・note・Instagramのアナリティクス画面から以下の数値を抽出する：いいね / 保存（ブックマーク）/ インプレッション / フォロワー増減 / コメント数。読み取れない項目は空欄のままにする。
4. 対応する下書きファイルを `Projects/投稿下書き/{プロジェクト}/` から探して「パフォーマンス記録」欄を更新する（見つからない場合はスキップ）
5. `Projects/投稿パフォーマンス.md` に以下の形式で**追記**する

**パス**: `Projects/投稿パフォーマンス.md`（Obsidian直接書き込み）

```markdown
| {YYYY-MM-DD} | {プロジェクト} | {タイトル/テーマ} | いいね:{n} / 保存:{n} / フォロワー:{+n} |
```

ファイルが存在しない場合は以下のヘッダーつきで新規作成する：

```markdown
# 投稿パフォーマンスログ

## RE:set

| 日付 | タイトル/テーマ | 数値 | 所感 |
|---|---|---|---|

## MATOI

| 日付 | タイトル/テーマ | 数値 | 所感 |
|---|---|---|---|

## AI情報発信

| 日付 | タイトル/テーマ | 数値 | 所感 |
|---|---|---|---|
```

#### 月次パターン抽出（毎月1日のみ）

毎月1日の日次クローズ時に、`Projects/投稿パフォーマンス.md` を読み込んで以下を自動抽出し、`Projects/投稿パフォーマンス.md` の末尾に追記する。

```markdown
## 勝ちパターン更新 — YYYY年MM月

### RE:set
- 効いたもの: {保存数上位のタイトル・共通点}
- 避けるべきもの: {反応が低かった傾向}

### MATOI
- 効いたもの: {同上}
- 避けるべきもの: {同上}

### AI情報発信
- 効いたもの: {同上}
- 避けるべきもの: {同上}
```

この内容はマーケティングエージェントがStep 0で読み込み、次回の生成に反映する。

- `[投稿記録]` がない日は Step 4b をスキップする

### Step 4c: 作業報告書の更新（平日毎日実行）

平日（月〜金・祝日除く）は必ず実行する。土日祝のみスキップ。

稼働時間を確認して xlsx を直接更新する。

以下を確認する：

```
Futuremind稼働時間を教えてください。
  開始時間（例: 9:30）
  終了時間（例: 18:30）
  休憩時間（デフォルト: 1:00。違う場合のみ教えてください）
```

回答を受け取ったら、Pythonスクリプト（Bashツール）で xlsx を更新する。

**ファイル構成**

| 項目 | 値 |
|---|---|
| ファイルパス | `/Users/kisinomasasi/projects/cc-company/agents/work-report/作業報告書_2026_04月.xlsx` ※月次で命名変更（`作業報告書_{YYYY}_{MM}月.xlsx`） |
| シート名 | `{YYYY}年{M}月`（例: `2026年4月`） |
| 対象行 | `5 + {当日の日付の日}` 番目の行（例: 4月7日 → 行12） |
| E列 | 開始時間（`datetime.time(H, M)` 形式） |
| F列 | 終了時間 |
| G列 | 休憩時間 |
| H列 | 作業内容（Step 0 の「今日やったこと」からFuturemind関連のみ抽出・整形） |

**実行コマンド（Bashツールで実行）**

```bash
python3 /Users/kisinomasasi/projects/cc-company/agents/work-report/scripts/update_work_report.py \
  "/Users/kisinomasasi/projects/cc-company/agents/work-report/作業報告書_{YYYY}_{MM}月.xlsx" \
  "{YYYY}年{M}月" {DAY} {H_START} {M_START} {H_END} {M_END} {H_BREAK} {M_BREAK} "作業内容テキスト"
# 出力: ok / error: ...
```

**注意事項**

- **月初（前月末からの月変わり）**: 対象月のシートが存在しない場合は先にシートを作成する
  ```bash
  python3 /Users/kisinomasasi/projects/cc-company/agents/work-report/scripts/create_monthly_sheet.py \
    "/Users/kisinomasasi/projects/cc-company/agents/work-report/作業報告書_{YYYY}_{MM}月.xlsx" {YYYY} {MM}
  # 出力: created: 2026年5月 / exists: 2026年5月 / error: ...
  ```
- ファイル自体が存在しない場合: `作業報告書（雛形）olive 2025.xlsx` をコピーして `作業報告書_{YYYY}_{MM}月.xlsx` を作成し、上記と同様にシートを追加する
- 既に当日行にデータがある場合: 上書きする前にユーザーに確認する

更新完了後、完了サマリー（Step 5）に以下を追加する：
```
📄 作業報告書更新: {YYYY}年{M}月 {MM}/{DD} 分を記録（{開始}〜{終了} / {稼働}h）
```

### Step 4d: Olive へ作業報告メール送信（月1回・月初第一営業日）

**送信タイミング**: その月の**第一営業日（土日祝除く最初の平日）**のみ実行。それ以外の日はスキップ。

前月分の xlsx を添付してまとめて送る。**xlsx の添付は必須・忘れ厳禁。**

以下の内容をユーザーに提示し、「OK」をもらってから `mcp__gmail__send_email` で送信する。

```

宛先: info@olive.style
CC: aki@olive.style, s_yamaguchi@olive.style
件名: 【作業報告書】{前月YYYY}年{前月MM}月分 / KISHINO MASASHI / Futuremind

　パス: /Users/kisinomasasi/projects/cc-company/agents/work-report/

本文:
お世話になっております。
{前月YYYY}年{前月MM}月分の作業報告書をお送りします。
```

- 送信は必ず「OK」をもらってから実行する
- 第一営業日の判定: `calendar.monthrange` 等で計算。祝日は固定祝日のみ考慮でOK

### Step 4e: 陣さんへの日報文章を生成（Futuremind稼働日のみ）

Step 4c で「あり」だった場合のみ実行する。

以下のフォーマットで日報テキストを生成してユーザーに提示する。ユーザーが陣さんに Slack で送る。

```
お疲れ様です。本日の報告です。

【本日の作業】
・（今日やったことからFuturemind関連を箇条書きで3〜5行に整形）

【明日の予定】
・（Step 0のQ4「明日やること」から簡潔に）

よろしくお願いします。
```

- 敬語・ビジネスライト（硬すぎず、くだけすぎず）
- 箇条書きは具体的な作業名を短く（「設計・実装」より「○○機能の実装」）
- 明日の予定がない場合は「【明日の予定】未定」とする
- テキストをそのままコピーできる形で出力する

## 品質基準
- 【日次報告】タグが見つからない場合は「Chatworkに [日次報告] タグ付きのメッセージを送ってください」と返す
- カレンダーイベントのタイトルはシンプルに（余計なプレフィックス不要）
- TODOの期日解釈は当年前提（1月より前の日付が出たら翌年扱い）
- Obsidianのノートが存在しない場合は新規作成する
